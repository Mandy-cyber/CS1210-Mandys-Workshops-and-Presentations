# CS1210: Making your GitHub Portfolio
> This is paired with the presentation [here](https://canva.link/cs1210-portfolio-building), and is meant to be engaged with alongside the live presentation or recording! 

## The Aim
The aim of this activity is for you to **customize and get creative with your GitHub profile**. You can think of your profile as your "developer homepage". It shows a technical audience (and even non-technical at times!) **who you are** and **what you've been working on**. It's a good starting point especially if you don't have a personal website, or are still working on building one out!

# Steps

## 1. Setup
1. Click "New Repository" on GitHub

2. Title it the _exact same_ as your GitHub username. **The exact same**. If your username is "Mandy-cyber" title it "Mandy-cyber"

3. Set the repo visibility to **'Public'**

4. Check the option to **"Add a README file"**

5. Click Create!! You should see a pop-up from GitHub saying that this is a :sparkles: special repo :sparkles:

<br>

## 2. Editing your `README`

### Context
Okay so now we have our starting point. The `README.md` file from earlier is what you want to put all the information you want displayed in. To work on it you can either clone your repository locally, or just work from the UI—it's upto you!

### Bare Minimum Info Needed

- [ ] Name
- [ ] University
- [ ] Major, Minor, Concentration
- [ ] Your role & company (if on co-op, or from previous experiences)
- [ ] Your technical skills and tools
- [ ] Contact information (e.g. linkedin)

<br>

## 3. Customizing (i.e. making it truly _yours_)

Okay so you probably now have a pretty boring document with the information above. How can we now make it more fun, more personalized, more **you** (while maintaining a level of professionalism as well)? Well, here are some ways:

### Skill Badges/Icons

- [shields.io](https://shields.io/) — custom badges
